# 道路巡检数据分析与报告系统 — 用户使用说明书

## 一、软件概述

道路巡检数据分析与报告系统是基于Web的道路巡检数据管理与分析平台，属于"基于无人机的道路巡检系统"（大学生创业训练项目）的核心软件组成部分。本系统用于接收和管理无人机采集的道路图像与GPS数据，通过计算机视觉算法自动检测道路病害（裂缝、坑洼、网状裂缝等），进行多维度统计分析，并自动生成标准化的巡检报告。

### 1.1 软件名称

道路巡检数据分析与报告系统 V1.0

### 1.2 开发环境

- 编程语言：Python 3.8+
- Web框架：Flask 2.3+
- 数据库：SQLite 3
- 图像处理：OpenCV 4.8+
- 数值计算：NumPy 1.24+
- 前端框架：Bootstrap 5 + Chart.js + Leaflet
- 运行环境：Windows / Linux / macOS

### 1.3 主要功能模块

1. 巡检数据导入：支持批量导入道路图像（JPG/PNG/BMP）和GPS轨迹数据（CSV）
2. 图像预处理：高斯去噪、CLAHE对比度增强、Canny边缘检测、形态学处理
3. 病害智能检测：双策略检测引擎（霍夫线检测裂缝 + 轮廓分析坑洼）
4. 数据管理：巡检记录存储、检索、删除
5. 统计分析：病害类型分布、严重程度分布、置信度统计、GPS轨迹分析
6. 地图可视化：GPS轨迹地图标注、病害位置标注
7. 报告生成：一键生成标准化HTML巡检报告（含养护建议）

## 二、安装与启动

### 2.1 环境准备

确保已安装 Python 3.8 或更高版本。

### 2.2 安装依赖

打开命令行终端，进入软件目录，执行：

```
pip install flask flask-sqlalchemy opencv-python numpy
```

如需YOLO深度学习检测功能，额外安装：

```
pip install onnxruntime
```

### 2.3 启动系统

在软件根目录下执行：

```
python app.py
```

启动后，打开浏览器访问 http://localhost:5000 即可进入系统。

## 三、操作指南

### 3.1 系统首页（仪表盘）

登录系统后进入首页仪表盘，可查看：
- 巡检任务总数、图像总数、病害总数
- 最近巡检记录列表
- 病害类型分布饼图
- 七日巡检趋势柱状图

通过左侧导航栏可访问各功能模块。

### 3.2 数据导入

点击左侧"数据导入"进入数据导入页面：

1. 填写巡检基本信息：
   - 任务名称（必填），例如"桂林-阳朔旅游公路2025年3月巡检"
   - 巡检日期（必填）
   - 巡检路段、巡检员、天气（选填）
   - 备注信息（选填）

2. 上传巡检图像：
   - 点击"选择文件"，可一次性选择多张道路巡检图像
   - 支持的格式：JPG、PNG、BMP
   - 建议分辨率≥1920×1080，单文件≤20MB

3. 上传GPS轨迹数据（可选）：
   - CSV格式，需包含列：lat/latitude, lng/longitude/lon
   - 可选列：alt, speed, timestamp
   - 示例格式见页面右侧说明

4. 点击"导入数据"按钮完成导入

### 3.3 巡检详情

导入成功后自动跳转至巡检详情页，可查看：
- 巡检基本信息
- 图像缩略图列表（已检测/未检测标注）
- 病害检测结果列表
- 可通过顶部按钮进入病害检测、统计分析、地图视图、生成报告

### 3.4 病害检测

在巡检详情页点击"病害检测"进入检测页面：

**单张检测：**
1. 在左侧图像列表中选择一张未检测图像
2. 系统自动执行检测（约2-5秒）
3. 右侧显示标注后的图像（标注框标注病害位置）
4. 下方显示检测到的病害详情表

**批量检测：**
1. 点击"批量检测所有未处理图像"按钮
2. 系统依次处理所有未检测图像
3. 完成后显示总结信息

**检测结果说明：**
- 蓝色标注框：霍夫线检测发现的裂缝
- 绿色标注框：轻微病害
- 黄色标注框：中等病害
- 红色标注框：严重病害

### 3.5 统计分析

在巡检详情页点击"统计分析"进入分析页面，包含：
- **病害类型分布**：饼图展示各类型病害数量占比
- **严重程度分布**：柱状图展示轻微/中等/严重病害数量
- **检测置信度分布**：高/中/低置信度分类统计
- **养护建议**：根据病害严重程度自动生成分级养护方案
- **GPS轨迹摘要**：轨迹点数、平均速度、中心坐标

### 3.6 地图视图

在巡检详情页点击"地图视图"进入地图页面：
- 蓝色线条：无人机飞行轨迹
- 绿色标记：轻微病害位置
- 橙色标记：中等病害位置
- 红色标记：严重病害位置
- 点击标记可查看病害详细信息

### 3.7 生成报告

点击"生成报告"按钮，系统自动生成标准化巡检报告（HTML格式），包含：
1. 巡检基本信息
2. 巡检数据概览（统计卡片）
3. 病害统计分析（类型/严重程度分布表）
4. 病害明细表
5. 养护建议（分紧急处理/计划修复/持续监测三级）
6. GPS轨迹摘要

点击"下载报告"可将报告保存为HTML文件。

### 3.8 巡检记录管理

点击左侧"巡检记录"进入记录列表：
- 按创建时间倒序展示所有巡检任务
- 支持分页浏览
- 每条记录可查看详情或生成报告

## 四、病害类型与严重程度说明

| 病害类型 | 描述 | 典型特征 |
|---------|------|---------|
| 横向裂缝 | 垂直于道路中心线方向的裂缝 | 宽高比 > 3 |
| 纵向裂缝 | 平行于道路中心线方向的裂缝 | 宽高比 < 0.33 |
| 网状裂缝 | 多条裂缝交织形成的网状损坏 | 多条线段交错，低填充比 |
| 坑洼 | 路面局部凹陷或破损 | 较规则形状，中高填充比 |

| 严重程度 | 等级 | 处理建议 |
|---------|------|---------|
| 轻微 | 1级 | 纳入日常巡查，持续监测 |
| 中等 | 2级 | 列入月度养护计划，密封处理 |
| 严重 | 3级 | 立即修补，必要时设置警示标志 |

## 六、注意事项

1. 首次运行系统会自动创建数据库文件（inspection.db），请勿手动删除
2. 图像文件存储在uploads目录，报告文件存储在reports目录
3. YOLO ONNX模型文件（road_defect.onnx）需放置于models目录方可启用深度学习检测
4. 系统默认使用OpenCV传统检测方法，准确率受图像质量影响
5. 建议定期清理uploads和reports目录以释放磁盘空间
6. GPS CSV文件编码需为UTF-8

## 七、技术特点

1. **双策略检测引擎**：霍夫变换线检测（针对裂缝）与轮廓分析（针对坑洼）协同工作
2. **模块化架构**：前后端分离，检测/分析/报告模块独立，便于扩展和维护
3. **自适应预处理**：CLAHE对比度增强+中值滤波+高斯去噪，适应不同光照条件
4. **智能分类**：基于形态学特征（宽高比、填充比、面积）的病害自动分类
5. **一键报告**：从数据到报告全流程自动化，输出标准化巡检文档
